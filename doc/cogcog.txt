*cogcog.txt*    LLM as a vim verb

INTRODUCTION                                    *cogcog*

cogcog adds LLM verbs to vim. They compose with motions and text
objects. Most verbs run at 0.3s via the bundled raw transport.
External checker commands are optional. Broader execute uses pi RPC.

Note: `ga` overrides show-ASCII, `gs` overrides sleep,
`<C-g>` overrides show-file-info.

FAST VERBS                                      *cogcog-verbs*

                                                *cogcog-ga*
ga{motion}      Explain code. No prompt. Count = verbosity:
                  gaip = concise, 1gaip = 1 sentence, 3gaip = detailed
                Auto-includes visible windows and quickfix.
                Workbench open = continues there. Closed = stateless.
                Visual: prompted.

                                                *cogcog-gaa*
gaa             Explain entire buffer.

                                                *cogcog-gs*
gs{motion}      Generate code. Raw API (fast). Output in code buffer.
                Includes workbench content when present.
                Auto-detects language, strips fences. :w to save.

                                                *cogcog-gss*
gss             Generate from entire buffer.

                                                *cogcog-gr*
<leader>gr{motion}
                Refactor in-place. Small rewrites apply directly
                (u to undo). Larger rewrites open a review buffer
                with a unified diff. Press 'a' to apply.

DEEPER / OPTIONAL VERBS                         *cogcog-cloud*

                                                *cogcog-gc*
<leader>gc{motion}
                Check code. By default uses the bundled cogcog
                transport. Set COGCOG_CHECKER to use a different
                command. Split reused.

                                                *cogcog-gx*
<leader>gx      pi RPC execute. Runs in the workbench.
                Uses visible state, quickfix, and workbench
                contents as prompt anchors. Starts
                `pi --mode rpc --no-session` by default.
                Set COGCOG_PI_RPC_CMD to override provider/model.
                Repeated use while pi is active sends a steering turn.
                UI requests use vim.ui and a temporary editor split.
                If `.cogcog/pi-bridge.sock` exists, Cogcog attaches to the
                companion harness instead of spawning pi directly.

USER COMMANDS                                    *cogcog-commands*

:CogcogHarness          Open an embedded harness terminal.
:CogcogCompanionStatus  Show socket/attach state.
:CogcogCompanionStop    Stop the companion broker.
:CogcogDetach           Detach the current local pi channel.

                                                *cogcog-cd*
<leader>cd      Discover project. Writes .cogcog/discovery.md
                (gf-navigable). By default uses the bundled cogcog
                transport. Set COGCOG_CHECKER to override.

PLANNING                                        *cogcog-plan*

                                                *cogcog-ctrl-g*
<C-g>           Fast workbench synthesis (raw API, 0.3s).
                From code: prompts, shows current filename.
                From workbench: sends the current workbench as-is.
                The model can request tool calls during synthesis.
                Each call appears in the workbench for approval.

                                                *cogcog-tool-mode*
                Set `vim.g.cogcog_tool_mode` to control approval:
                  "ask"   (default) approve every tool call
                  "read"  auto-approve read-only, ask for commands
                  "trust" auto-approve all for this turn

                                                *cogcog-cy*
<leader>gy      (visual) Pin selection to workbench.
<leader>g!      Execute a shell command; output streams to the workbench.
                Opens workbench automatically. Cancellable with <C-c>.
<leader>ct      Pick and run a project tool from `.cogcog/tools/`.
                Output streams to the workbench. Tools are plain scripts.
<leader>cT      Generate a new tool. Describe what you need, review the
                generated script, press `a` to save to `.cogcog/tools/`.

VIM STATE AS CONTEXT                            *cogcog-context*

                                                *cogcog-gj*
<leader>gj      Jump trail. Last 8 locations. Reuses ask split.

                                                *cogcog-g-dot*
<leader>g.      Recent changes. Reuses ask split.

                                                *cogcog-gq*
<leader>gq      Summarize the current quickfix target set.

                                                *cogcog-gQ*
<leader>gQ      Review and prioritize the current quickfix target set.

                                                *cogcog-gR*
<leader>gR      Prepare rewrites for the current quickfix target set.
                Opens a review buffer with diffs. Press 'a' to apply
                all prepared rewrites. Targets that changed since
                review are skipped.

ga auto-includes visible windows and quickfix.

Scope model:
    hard scope      explicit operand or quickfix target set
    explicit import workbench contents and pinned snippets
    soft context    visible windows for grounding only

Quickfix is the deliberate batch boundary when present.
Visible windows help grounding, but should not replace explicit scope.

WORKBENCH                                       *cogcog-workbench* *cogcog-panel*

                                                *cogcog-co*
<leader>co      Toggle workbench. Open = ga can continue there.
>
                :read .cogcog/review.md     add skill
                :read !git diff             add tool output
                dap                         delete section
<
                                                *cogcog-cc*
<leader>cc      Clear workbench and persisted workbench file.

                                                *cogcog-cp*
<leader>cp      Improve prompt from bad response.

                                                *cogcog-cancel*
<C-c>           Cancel jobs. Pass-through when idle.

CONFIGURATION                                   *cogcog-config*

Default behavior:
    ga/gs/gr/<C-g>/gc/cd use the bundled cogcog command.
    No external harness is required for the core loop.

Optional overrides:
    gx              COGCOG_PI_RPC_CMD (explicit pi RPC command)
    gc/discover     COGCOG_CHECKER (explicit stronger review/discovery command)

                                                *cogcog-fast-model*
COGCOG_FAST_MODEL   Fast model for most verbs via the bundled cogcog command.

                                                *cogcog-pi-rpc-cmd*
COGCOG_PI_RPC_CMD   Optional pi RPC command for <leader>gx.
                Default: `pi --mode rpc --no-session`.
                Use it to pick a specific provider/model.

                                                *cogcog-checker*
COGCOG_CHECKER      Optional stronger command for check/discover.
                When unset, check/discover fall back to cogcog --raw.

                                                *cogcog-pi-socket*
COGCOG_PI_SOCKET    Optional companion harness socket path.
                Default: `.cogcog/pi-bridge.sock`.
                When present and connectable, gx / workbench pi-RPC flows
                attach to the shared harness session.

                                                *cogcog-backend*
COGCOG_BACKEND      "anthropic", "openai", "codex", "copilot", or "pi"
                (default: anthropic)
                codex: reads pi's Codex OAuth, calls API with curl (18ms).
                copilot: reads pi's Copilot OAuth, calls API with curl (14ms).
                  smart model: claude-opus-4.6
                  fast model (--raw): claude-sonnet-4.6
                pi: delegates to `pi -p` (~0.7s startup).
                See |cogcog-pi-provider|.

                                                *cogcog-pi-provider*
COGCOG_PI_PROVIDER  Provider for pi backend (default: openai-codex).
                Any provider pi supports: anthropic, google, etc.

                                                *cogcog-api-url*
COGCOG_API_URL      API endpoint (anthropic/openai backends only).

                                                *cogcog-api-key*
COGCOG_API_KEY      API key (anthropic/openai backends only).

                                                *cogcog-model*
COGCOG_MODEL        Default model when FAST_MODEL not set.

                                                *cogcog-max-tokens*
COGCOG_MAX_TOKENS   Max tokens (default 8192).

PROJECT FILES                                   *cogcog-project*

.cogcog/system.md       System prompt. Shared by all verbs.
.cogcog/workbench.md    Auto-saved/restored workbench.
.cogcog/discovery.md    Project map.
.cogcog/pi-bridge.sock  Optional companion harness socket.

Companion harness:
>
    bin/cogcog-bridge   start shared pi RPC broker
    bin/cogcog-harness  start/attach terminal harness client
    :CogcogHarness      open embedded harness terminal
<
