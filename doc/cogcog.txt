*cogcog.txt*    Cogcog Neovim bridge for pi

INTRODUCTION                                    *cogcog*

cogcog turns vim motions and selections into structured events for pi.
It is a Neovim-side operator layer plus a small RPC bridge.

Typical setup: >
    Terminal 1: nvim
    Terminal 2: pi
    /cogcog-claim
<

With the bundled pi extension, these event types are forwarded to pi:
    ask, generate, refactor, check, plan, execute

Every event is also exposed locally via |User| `CogcogEvent` and
`vim.g.cogcog_last_event`, so custom listeners can build on top.

FAST VERBS                                      *cogcog-verbs*

                                                *cogcog-ga*
ga{motion}      Emit an ask / explain event for the scoped text.
                Count changes the built-in ask wording:
                  gaip = normal
                  1gaip = shorter ask
                  3gaip = more detailed ask
                Visual `ga` prompts for an explicit question.

                                                *cogcog-gaa*
gaa             Emit an ask / explain event for the whole buffer.

                                                *cogcog-gs*
gs{motion}      Emit a generate event for the scoped text.
                Visual `gs` prompts for an instruction.

                                                *cogcog-gss*
gss             Emit a generate event for the whole buffer.

                                                *cogcog-gr*
<leader>gr{motion}
                Emit a refactor event for the scoped text.
                The payload includes exact target file + line range.
                Visual mode refactors the selected range.

                                                *cogcog-gc*
<leader>gc{motion}
                Emit a check / review event for the scoped text.
                Visual mode checks the selection.

PLANNING                                        *cogcog-plan*

                                                *cogcog-ctrl-g*
<C-g>           Emit a plan / continue event.
                From a normal file, prompts for a short request.
                From the workbench, means “continue from here”.

                                                *cogcog-gx*
<leader>gx      Prompt in Neovim, then emit an execute event to pi.
                Use this for direct do-work instructions.

WORKBENCH                                       *cogcog-workbench* *cogcog-panel*

cogcog keeps a persistent scratch buffer named `[cogcog-workbench]`.
Use it to collect snippets, write notes, and anchor longer pi turns.

                                                *cogcog-co*
<leader>co      Toggle the workbench window.

                                                *cogcog-cy*
<leader>gy      In visual mode, append the selection to the workbench.

                                                *cogcog-cc*
<leader>cc      Clear the workbench and delete `.cogcog/workbench.md`.

Workbench state persists across editor exits in: >
    .cogcog/workbench.md
<

PI INTEGRATION                                  *cogcog-pi*

cogcog auto-starts a Neovim server socket on load: >
    /tmp/cogcog.sock        (or COGCOG_NVIM_SOCKET)
<

Install the bundled pi extension: >
    cd /path/to/cogcog/pi-extension
    npm install
    ln -s /path/to/cogcog/pi-extension ~/.pi/agent/extensions/cogcog
<

Inside pi: >
    /reload
    /cogcog-claim
    /cogcog-status
<

Only the claimed pi session receives live Cogcog events.

The extension injects current editor state before each Cogcog-triggered turn:
    - current buffer + cursor
    - visible windows
    - quickfix list
    - diagnostics summary
    - nearby lines around the cursor

It also registers these tools for pi:
    nvim_context
    nvim_buffer
    nvim_buffers
    nvim_diagnostics
    nvim_goto
    nvim_quickfix
    nvim_exec
    nvim_notify

If no pi session is listening, cogcog warns in Neovim instead of silently
pretending work happened.

LOCAL EVENT HOOKS                               *cogcog-events*

Every emit also triggers a local Neovim event: >
    :autocmd User CogcogEvent lua print(vim.inspect(vim.v.event.data))
<

The last emitted payload is also stored in: >
    vim.g.cogcog_last_event
<

You do not need this for the normal bundled workflow, but it is available
for custom listeners and automations.

SHELL FILTER                                    *cogcog-shell* *cogcog-backend*

This repo also ships `bin/cogcog`, a separate stdin -> LLM -> stdout helper.
It is independent from the Neovim event flow.

Examples: >
    echo "explain this" | cogcog
    cat src/main.ts | cogcog --raw
    git diff --staged | cogcog "review this"
<

CONFIGURATION                                   *cogcog-config*

Neovim / bridge config:

                                                *cogcog-nvim-socket*
COGCOG_NVIM_SOCKET  Neovim server socket path for the pi bridge.
                Default: `/tmp/cogcog.sock`.

Shell helper config (`bin/cogcog`):

                                                *cogcog-cmd*
COGCOG_CMD      Delegate `cogcog` to another CLI.

                                                *cogcog-api-url*
COGCOG_API_URL      Default API endpoint.

                                                *cogcog-fast-api-url*
COGCOG_FAST_API_URL API endpoint for `--raw` fast path.

                                                *cogcog-api-key*
COGCOG_API_KEY      API key.
OPENAI_API_KEY      Fallback API key.

                                                *cogcog-model*
COGCOG_MODEL        Default model name.

                                                *cogcog-fast-model*
COGCOG_FAST_MODEL   Model name for `--raw`.

                                                *cogcog-max-tokens*
COGCOG_MAX_TOKENS   Max tokens. Default: 8192.

                                                *cogcog-system*
COGCOG_SYSTEM       Optional system prompt for `bin/cogcog`.

PROJECT FILES                                   *cogcog-project*

.cogcog/workbench.md    Persisted workbench scratchpad.
lua/cogcog/init.lua     Keymaps and event emission.
lua/cogcog/context.lua  Scope builders and workbench helpers.
lua/cogcog/transport.lua  Event emit + `User CogcogEvent`.
lua/cogcog/bridge.lua   RPC surface used by the pi extension.
pi-extension/           Bundled pi extension.
bin/cogcog              Shell filter.

QUICK REFERENCE                                 *cogcog-cheatsheet*

Bundled pi bridge: >
    ga{motion} / gaa    ask / explain
    Visual ga           ask question about selection
    gs{motion} / gss    generate
    <leader>gr{motion}  refactor
    <leader>gc{motion}  check / review
    <C-g>               plan / continue
    <leader>gx          execute prompt -> pi
    <leader>gy          pin to workbench
    <leader>co          toggle workbench
    <leader>cc          clear workbench
<

pi commands: >
    /cogcog-claim
    /cogcog-release
    /cogcog-status
<

Internal hook: >
    User CogcogEvent
    vim.g.cogcog_last_event
<
