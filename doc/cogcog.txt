INTRODUCTION                                    *cogcog*

cogcog adds three LLM verbs to vim: ask, generate, check. They compose
with motions and text objects like any vim operator.

Two context modes: you curate (ga) or the LLM curates (plan/gs/gc).

Note: `ga` overrides vim's built-in show-ASCII, `gs` overrides
sleep, `<C-g>` overrides show-file-info. These are deliberate
tradeoffs for frequently-used LLM verbs.

VERBS                                           *cogcog-verbs*

                                                *cogcog-ga*
ga{motion}      Ask about code. Fast local model. You provide the
                context via selection/motion.

                Stateless by default (throwaway split, reused).
                If the context panel is open, becomes stateful.

                Quickfix entries are auto-included as context.
                Response split closes with `q`.

                                                *cogcog-gs*
gs{motion}      Generate code. Agent backend with tool calling.
                Can read files, run commands, search the web.
                Output goes to a fresh code buffer with line numbers.

                Auto-detects language from code fences.
                Use `:w filename` to save. Close with `q`.
                Planning context is included if non-empty.

                                                *cogcog-gc*
<leader>gc{motion}
                Check code with strongest model (Opus by default).
                Deep review for correctness, edge cases, bugs.
                Response split closes with `q`.

PLANNING                                        *cogcog-plan*

                                                *cogcog-ctrl-g*
<C-g>           Agentic planning. Uses agent backend — can read
                files, search the codebase, use tools. The LLM
                decides what context is relevant.

                From context panel: sends buffer as-is.
                From any other buffer: prompts for question.

                Pin specific code with <leader>cy when you know
                better than the LLM what's relevant.

                                                *cogcog-cy*
<leader>cy      (visual) Pin selection to context panel. Manual
                context curation. Does not send.

CONTEXT PANEL                                   *cogcog-panel*

                                                *cogcog-co*
<leader>co      Toggle the context panel. When open, `ga` becomes
                stateful — questions and answers accumulate.

                The panel is a regular buffer. Use native vim:
>
                :read .cogcog/review.md     add a skill
                :read !git diff             add tool output
                dap                         delete a section
<
                                                *cogcog-cc*
<leader>cc      Clear context and reload system prompt.

                                                *cogcog-cancel*
<C-c>           Cancel any running cogcog job. Passes through to
                default <C-c> when no jobs are active.

DISCOVERY                                       *cogcog-discover*

                                                *cogcog-cd*
<leader>cd      Discover project. Uses strongest model with tool
                access to read files and map the codebase by domain.

                Output saved to `.cogcog/discovery.md` with
                gf-navigable file paths.

                Three options when discovery exists:
                - Open: view existing map
                - Update: incremental refresh
                - Re-discover: full re-analysis

PROMPT IMPROVEMENT                              *cogcog-improve*

                                                *cogcog-cp*
<leader>cp      Improve prompts from bad responses. Navigate to
                the bad response, press <leader>cp, describe what
                was wrong. Generates a system prompt fix and
                appends to `.cogcog/system.md`.

CONFIGURATION                                   *cogcog-config*

Three independent backend paths:

    ga          COGCOG_FAST_MODEL via COGCOG_API_URL
    gs/<C-g>    COGCOG_CMD (agent with tools)
    gc/discover COGCOG_CHECKER (strongest model)

                                                *cogcog-fast-model*
COGCOG_FAST_MODEL   Model for `ga` ask. Used with --raw.

                                                *cogcog-cmd*
COGCOG_CMD          Agent CLI for `gs` and `<C-g>`.
                    Must be stdin→stdout.

                                                *cogcog-checker*
COGCOG_CHECKER      CLI for `gc` and `<leader>cd`.
                    Default: pi -p --provider anthropic --model opus:xhigh

                                                *cogcog-backend*
COGCOG_BACKEND      API type: "anthropic" (default) or "openai"

                                                *cogcog-api-url*
COGCOG_API_URL      API endpoint for raw path.

                                                *cogcog-api-key*
COGCOG_API_KEY      API key.

PROJECT FILES                                   *cogcog-project*

.cogcog/system.md       System prompt. Auto-loaded. Improved via <leader>cp.
.cogcog/session.md      Auto-saved on exit, restored on open.
.cogcog/discovery.md    Project map from <leader>cd.
.cogcog/*.md            Skill templates. `:read` into context.

SHELL USAGE                                     *cogcog-shell*
>
    echo "explain CRDs" | cogcog
    git diff --staged | cogcog --raw "review this"
<
--raw forces direct API (bypasses COGCOG_CMD, uses COGCOG_FAST_MODEL).

QUICK REFERENCE                                 *cogcog-cheatsheet*

Verbs (you curate context): >

    ga{motion}          ask (fast, local)
    gs{motion}          generate (agent, tools)
    <leader>gc{motion}  check (deep, strongest)
<

Context & planning (LLM curates context): >

    <C-g>            plan (agentic, reads files)
    <leader>cy       pin selection to context
    <leader>co       toggle context panel
    <leader>cd       discover / update project map
    <leader>cp       improve prompt from bad response
    <leader>cc       clear context
    <C-c>            cancel running job
    q                close response split
<

Combos: >

    gd → gaf                    definition → ask
    :make → gaip                errors → explain
    ggVG ga                     explain entire file
    <leader>cy × N → <C-g>     pin from files → plan
    gsip → <leader>gcip         generate → verify
    <leader>cd → <C-g>          discover → explore
<
