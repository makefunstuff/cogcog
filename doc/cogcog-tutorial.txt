*cogcog-tutorial.txt*   Getting started

SETUP                                           *cogcog-tutorial-setup*
>
    cd /path/to/cogcog/pi-extension && npm install
    ln -s /path/to/cogcog/pi-extension ~/.pi/agent/extensions/cogcog
<
Inside pi: >
    /reload
    /cogcog-claim
    /cogcog-status
<
Neovim: open any code file and try: >
    gaip
<
If no pi session is listening, Neovim warns instead of hiding it.

ASK / EXPLAIN                                   *cogcog-tutorial-ask*
>
    gaip            ask about inner paragraph
    gaa             ask about entire buffer
    3gaip           more detailed ask wording
    Visual ga       prompted question about selection
<
These emit an `ask` event to the claimed pi session.

GENERATE                                        *cogcog-tutorial-gen*
>
    gsip            generate from scoped text
    gss             generate from whole buffer
    Visual gs       generate from selection
<
These emit a `generate` event.

REFACTOR                                        *cogcog-tutorial-refactor*
>
    <leader>grip    refactor scoped text
    Visual <leader>gr
<
The event includes exact target file + line range for pi.

CHECK                                           *cogcog-tutorial-check*
>
    <leader>gcaf
    Visual <leader>gc
<
These emit a `check` event for correctness / edge-case review.

EXECUTE / PLAN / WORKBENCH                      *cogcog-tutorial-plan*
>
    <leader>gx      execute prompt -> pi
    <leader>co      open workbench
    <leader>gy      pin selection
    <C-g>           plan / continue
    <leader>cc      clear workbench
<
Workbench persists in `.cogcog/workbench.md`.

PI BRIDGE                                       *cogcog-tutorial-pi*
>
    Terminal 1: nvim
    Terminal 2: pi
<
The bundled extension injects live editor state into pi and registers:
>
    nvim_context
    nvim_buffer
    nvim_buffers
    nvim_diagnostics
    nvim_goto
    nvim_quickfix
    nvim_exec
    nvim_notify
<

INTERNAL EVENT HOOK                             *cogcog-tutorial-events*
>
    User CogcogEvent
    vim.g.cogcog_last_event
<
The bundled pi extension forwards ask/generate/refactor/check/plan/execute.
The hook is there if you ever want a custom listener later.

SHELL                                           *cogcog-tutorial-shell*
>
    export COGCOG_API_URL=http://localhost:8091/v1/chat/completions
    export COGCOG_MODEL=gemma4:26b
    export OPENAI_API_KEY=dummy
    echo "explain this diff" | cogcog
    cat src/main.ts | cogcog --raw
<
